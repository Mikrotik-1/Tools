Troubleshooting & Solutions

A comprehensive guide to the most common technical problems facing MikroTik users with practical and guaranteed solutions

Most Common Problems Solutions

Problem: There is an IP sending messages inside the network - when these messages are many, they create a loop in the network and cause abnormal things:

  • Some clients appear online but have no internet
  • Service is very slow for them
  • General service instability

How do I know if there's an IP inside my network?

Three detection methods:
First Method

DHCP Client

Second Method

ARP

Third Method

Tools ‚Üí IP Scan

Solution:

  1. Step One: Try to access this device and disable its DHCP server.

    If unable to connect:

  2. Step Two: Try to locate the device by dividing the network into areas and disconnecting each area.

    If the IP disappears, it will be easy to reach the device and disconnect it from your network.

Warning: Leaving the device in the network will cause service instability.

Problem: This happens due to choosing an inappropriate Subnet for your network's IP count. (Example: /24)

Solution:

Provided by our site in the settings:

Choose a larger Subnet:
/16 /8 /4

There will be plenty of public IPs.

How do I know my device's usage strength?

Tools ‚Üí Profile

Result analysis: If there is high usage or "IDE" keyword exceeds 50%, this means the device cannot process data quickly, especially with the evolution of websites like AI.

Suggested Solutions:

First Solution: Update the version

If you're using a CPU with version 5, update to version 6.

Second Solution: Change hardware

If your router board is weak, change it to a better router that suits your network.
